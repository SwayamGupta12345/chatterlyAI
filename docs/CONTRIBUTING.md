# Contributing Guidelines

## Branch format
- `feat/<name>`
- `fix/<bug>`
- `docs/<change>`

## Commit messages
Follow:
<type>(scope): message

Examples:
- feat(api): add conversation route
- fix(frontend): resolved typing issue

## How to contribute
1. Open an Issue
2. Fork the repo
3. Create a branch
4. Make changes
5. Submit a PR

## Where to contribute
- UI → this repo
- Business logic → backend repos
- Agent workflows → agentic repo
